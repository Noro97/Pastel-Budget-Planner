## 2024-04-22 - Optimizing Date Sorting for ISO strings
**Learning:** Instantiating `Date` objects inside array sort callbacks `(a, b) => new Date(a.date).getTime() - new Date(b.date).getTime()` is extremely slow for large datasets. For dates consistently formatted as ISO strings (`YYYY-MM-DD`), standard string comparison or `localeCompare()` yields identical chronologically sorted results but runs nearly ~10x faster because it avoids the overhead of date parsing and object instantiation inside the loop.
**Action:** When strictly checking equality or sorting between two dates formatted consistently as 'YYYY-MM-DD' ISO strings, use direct string comparison (`localeCompare` or standard operators) instead of converting them to `Date` objects to avoid unnecessary performance overhead.
